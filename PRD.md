## 一、背景与目标

### 1.1 背景

瑞士德语区（Zürich）的中小学生在完成家庭作业时，经常面临以下问题：

1. **即时性**：课后遇到不会的题目，家长不一定懂，私人家教需预约，等待时间长
2. **成本**：Zürich 的专业家教价格高昂（CHF 60–120/小时），普通家庭难以负担全天候服务
3. **透明度**：家长很难直观了解孩子在哪些知识点上存在薄弱环节
4. **质量控制**：纯 AI 答疑存在给出错误答案、直接给答案而非引导思考的风险

### 1.2 产品目标

构建一个以 **AI 为主、真人老师兜底** 的学习支持平台，核心价值主张：

- **学生**：课后随时提问，获得分步引导式解答，不被卡住
- **家长**：以订阅价格获得接近家教质量的服务，透明可追踪
- **老师**：专注处理真正有难度的问题，减少重复劳动，按质量计酬

### 1.3 范围（MVP）

| 维度 | 范围 |
|------|------|
| 地区 | 瑞士德语区（Zürich 为主） |
| 学段 | Sekundarschule（7–9 年级）+ Gymnasium |
| 学科 | **仅数学**（MVP 阶段） |
| 语言 | 德语（UI）+ 英语（部分） |
| 平台 | Web 端（PWA，移动端友好） |

## 二、用户角色

### 2.1 学生（Student）

**画像**：12–19 岁，每天完成作业时可能需要帮助，使用手机或平板较多

**核心旅程**：
```
发现 STOA → 注册账号（或家长注册）
→ 上传/输入题目（文字 + 拍照）
→ 收到 AI 分步解答
→ [仍不理解] → 请求老师帮助
→ 老师接管回答
→ 标记为已解决
→ 继续下一题
```

**关键需求**：
- 操作简单，提问要快（< 30 秒完成提交）
- AI 解答是分步引导，不是直接给答案
- 等待老师时有状态提示（老师响应 SLA）
- 历史记录可查看

### 2.2 家长（Parent）

**画像**：30–50 岁，关注孩子学业，为教育付费的决策者

**核心旅程**：
```
通过广告/口碑了解 STOA
→ 为孩子注册（或批准孩子注册）
→ 选择订阅套餐
→ 查看孩子每周学习摘要
→ 看到薄弱知识点报告
→ 决定续订
```

**关键需求**：
- 每周一份自动生成的学习报告（邮件发送）
- 清晰的知识点薄弱分析（哪些知识点反复出错）
- 老师接管情况记录（看到有人兜底）
- 订阅管理（升降级、取消）

### 2.3 老师（Teacher）

**画像**：大学生、退休教师、兼职家教，希望灵活工作

**核心旅程**：
```
通过 STOA 招募入驻
→ 查看待处理题目队列
→ 接管题目（锁定，避免重复处理）
→ 查看 AI 已有的解答上下文
→ 补充/修正/深度解释
→ 标记解决
→ 按质量/数量获得报酬
```

**关键需求**：
- 工作台清晰，无需学习成本
- 看到 AI 已给了什么，知道从哪里接
- 快速回复工具（富文本 + 公式支持）
- 响应时间追踪（SLA 达标率影响报酬）

### 2.4 管理员（Admin）

**画像**：STOA 运营团队，1–3 人

**核心需求**：
- 用户管理（手动修改订阅等级，处理异常账号）
- 平台数据总览（DAU/MAU、AI 满意度、老师响应时效）
- 内容安全审核（查看举报内容）

## 三、功能需求

### 3.1 认证系统

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 邮箱注册 | 支持学生/家长/老师三种角色注册 | P0 |
| 邮件验证 | 注册后发送验证邮件 | P0 |
| 邮箱密码登录 | 标准登录流程 | P0 |
| JWT 刷新 | AccessToken 1小时过期，RefreshToken 30天 | P0 |
| 忘记密码 | 邮件重置链接 | P1 |
| 家长-学生绑定 | 家长注册时关联孩子账号 | P1 |

### 3.2 题目提交与 AI 答疑

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 文字提问 | 直接输入题目文字（支持 LaTeX 数学符号） | P0 |
| 图片上传 | 拍照上传作业图片（最大 10MB） | P0 |
| OCR 提取 | 自动识别图片中的题目文字 | P0 |
| OCR 校正 | 学生可编辑 OCR 结果后再提交 | P1 |
| 年级/学科选择 | 选择年级 + 学科（MVP 仅数学） | P0 |
| AI 分步解答 | Claude Haiku 生成分步引导式解答（非直接给答案） | P0 |
| 解答反馈 | 学生对 AI 解答评分（1–5 星） | P1 |
| 每日限额 | Free: 5次/天，Standard: 30次/天，Premium: 无限制 | P0 |
| 处理状态 | pending → ai_answering → ai_answered → teacher_requested → teacher_active → resolved | P0 |

### 3.3 老师接管

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 请求老师 | AI 回答后，学生可点击「请求老师帮助」 | P0 |
| 老师队列 | 老师工作台显示待处理题目列表（FIFO） | P0 |
| 题目锁定 | 老师接管后题目锁定（避免重复处理） | P0 |
| AI 上下文展示 | 老师看到题目 + AI 已给的解答 | P0 |
| 老师回复 | 富文本回复（支持公式） | P0 |
| AI 自动静默 | 老师接管后 AI 不再主动回复 | P0 |
| 标记解决 | 老师/学生均可标记为已解决 | P0 |
| 响应时间追踪 | 从请求到老师接管的时间（SLA 监控） | P1 |

### 3.4 家长面板与周报

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 孩子学习摘要 | 本周提问数、AI 解决率、薄弱知识点 Top 3 | P1 |
| 知识点趋势图 | 过去 4 周的知识点掌握变化 | P1 |
| 老师接管记录 | 本周老师接管详情 | P1 |
| 自动周报生成 | 每周一自动生成 PDF 风格 HTML 报告 | P1 |
| 周报邮件发送 | 每周一 8:00（Zürich 时间）发送到家长邮箱 | P1 |
| 订阅管理 | 查看当前套餐（Phase 2 接入 Stripe） | P2 |

### 3.5 管理后台

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 用户列表 | 分页查询所有用户 + 搜索 | P1 |
| 订阅管理 | 手动修改用户订阅等级 | P1 |
| 平台统计 | DAU/MAU、AI 答疑量、老师响应时效 | P1 |
| 内容审核 | 查看被举报/异常内容 | P2 |

## 四、非功能需求

### 4.1 性能

| 指标 | 目标 |
|------|------|
| 题目提交到 AI 开始回复 | < 3 秒（P90） |
| AI 完整回复生成 | < 30 秒（P90） |
| OCR 识别完成 | < 5 秒（P90） |
| 页面首次加载（FCP） | < 1.5 秒（4G 网络） |
| API 一般请求延迟 | < 200ms（P99，非 AI 请求） |

### 4.2 可用性

| 指标 | 目标 |
|------|------|
| 整体可用性 | 99.5%（月度） |
| 计划内停机 | 每次 < 30 分钟，提前 24 小时通知 |
| Lambda 并发 | 默认 10 并发，按需扩展 |

### 4.3 安全

- 所有 API 需 JWT 认证（除 /health、/auth/* ）
- S3 无公开访问，所有资源通过预签名 URL 或 CloudFront OAC 访问
- 用户数据严格隔离（DynamoDB 查询始终带 user_id 过滤）
- Bedrock Prompt 注入防护（清理用户输入中的 system prompt 关键词）
- WAF 速率限制：同一 IP 每分钟不超过 60 次请求
- GDPR/nDSG 合规（瑞士数据保护法）：数据存储在 eu-central-2（苏黎世）

### 4.4 可观测性

- 所有 Lambda 日志发送至 CloudWatch Logs
- 结构化 JSON 日志（包含 request_id, user_id, duration_ms）
- CloudWatch Dashboard：AI 调用量、错误率、P99 延迟
- 告警：错误率 > 1%（5分钟窗口）触发 SNS 通知

## 五、订阅与商业模式

### 5.1 套餐详情

| 套餐 | 月费 | AI 答疑次数 | 老师服务 | 周报 |
|------|------|------------|---------|------|
| Free | CHF 0 | 5 次/天 | 无 | 无 |
| Standard | CHF 39–49 | 30 次/天 | 文字答疑（优先队列） | ✅ |
| Premium | CHF 79–99 | 无限制 | 优先响应（< 1小时） + 详细报告 | ✅（增强版） |

### 5.2 MVP 阶段付费方式

MVP 阶段不集成支付系统，付费流程：
1. 家长通过官方渠道（邮件/WhatsApp）联系 STOA
2. 手动开通订阅（管理员后台修改 subscription_tier）
3. 转账至 STOA 账户

Phase 2 接入 Stripe Subscription，支持信用卡/TWINT。

## 六、成功指标

### 6.1 MVP 核心指标（上线 8 周后）

| 指标 | 目标值 |
|------|--------|
| 付费订阅家庭数 | ≥ 10 |
| 学生 7 日留存率 | ≥ 40% |
| AI 答疑满意度（平均星级） | ≥ 3.5/5 |
| 老师接管率（占总提问比例） | ≤ 30% |
| 老师平均响应时间 | ≤ 4 小时 |
| 月 AWS 费用 | ≤ CHF 150 |

### 6.2 关键假设

1. **家长付费意愿**：Zürich 家长愿意为 CHF 39–49/月 的订阅付费（对比家教 CHF 60–120/小时）
2. **AI 质量**：Claude Haiku 对德语中学数学题的解答质量足够好（满足度 ≥ 3.5 星）
3. **老师供给**：能招募 3–5 名兼职老师，覆盖学生提问高峰（16:00–21:00 Zürich 时间）
4. **OCR 准确率**：Rekognition 对德语数学题手写作业的识别率 ≥ 70%
